# Prioritization Frameworks

## Why Prioritize
There is always more to build than capacity allows. Prioritization frameworks make
trade-offs explicit and defensible, so a team spends its limited effort on the work
with the highest expected value.

## RICE
RICE scores each item by Reach, Impact, Confidence, and Effort. Reach is how many
users are affected in a period; Impact is how much it moves the goal per user (often
scored 0.25 to 3); Confidence is a percentage discount for uncertainty; Effort is
person-months. The score is (Reach x Impact x Confidence) / Effort. RICE is good
when you have data and want a comparable numeric ranking.

## MoSCoW
MoSCoW sorts requirements into Must-have, Should-have, Could-have, and Won't-have
(this time). It is fast and communicates scope boundaries clearly, which makes it
useful inside a single release. Its weakness is that everything tends to drift into
"Must," so enforce a cap on the Must list.

## Kano Model
The Kano model classifies features by how they affect satisfaction: basic
expectations (their absence angers users, their presence is unnoticed), performance
features (more is better, linearly), and delighters (unexpected features that create
outsized delight). It helps balance table-stakes work against differentiation.

## Value vs. Effort (2x2)
Plotting items on a value-versus-effort grid quickly surfaces quick wins (high
value, low effort) and money pits (low value, high effort). It is a fast, visual
first pass, though "value" and "effort" are estimates.

## Weighted Scoring
For complex decisions, define several weighted criteria (revenue, strategic fit,
risk, effort) and score each option. It is flexible but only as good as the weights,
which should be agreed up front to avoid rationalizing a favored answer.

## Choosing a Framework
Use MoSCoW or a 2x2 for speed, RICE or weighted scoring when you need rigor and have
data, and Kano when balancing satisfaction types. The framework is a tool for a
conversation, not a substitute for judgment.
